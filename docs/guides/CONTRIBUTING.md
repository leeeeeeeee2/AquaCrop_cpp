# Docs Contributing Guide

This guide covers contributing to the documentation for AquaCrop-cpp.

Overview
- Write clear, concise docs with examples where helpful.
- Keep API references synchronized with public headers under include/.
- When making changes to code and docs in parallel, provide cross-references in commits.

Getting Started
- Ensure you can build the project: follow the Quick Start in README.md.
- Run any existing docs build steps (if provided) or generate docs using your preferred tools.

Documentation Style
- Use ASCII by default; preserve existing project typography.
- Use consistent headings and short paragraphs; add code blocks for examples.

Submitting Changes
- Open a pull request with a descriptive title and a short rationale.
- Include links to related issues or discussion threads if any.
- Update docs/api.md or docs/design.md where relevant.
